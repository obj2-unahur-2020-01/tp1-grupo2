Un poco de Historia


En julio de 2011 JetBrains reveló el Proyecto Kotlin, un nuevo lenguaje para la JVM, el cual había estado bajo desarrollo por un año.4​ El líder de JetBrains, Dmitry Jemerov, dijo que la mayoría de lenguajes no tienen las características que buscaban, con la excepción de Scala. Sin embargo, citó el lento tiempo de compilación de Scala como una deficiencia obvia. Uno de los objetivos establecidos de Kotlin es el de compilar tan rápido como Java.

En febrero de 2012, JetBrains liberó el código fuente del proyecto bajo la Licencia Apache 2.5​. Jetbrains espera que el nuevo lenguaje impulsará las ventas de IntelliJ IDEA.6​En 2017, recibe un importante impulso al ser nombrado por Google como lenguaje oficial para Android al mismo nivel que Java.7​Slide 1

comparación de código Clases

Para crear una clase en Kotlin usamos la palabra reservada class al igual que los demás lenguajes,

En Kotlin las clases son publicas por defecto y no es necesario colocarlo a diferencia de Java.


	//En java
	public class Car{
	//código
	}


	//En Kotlin
	class Car{
	// código
	}

comparación de código Propiedades

Las propiedades en Kotlin son muy fáciles de definir, solo basta con especificar si serán mutable (var) o inmutables (val) 


	//En java
	public class Car{
	String brand;
	String model;
	int year;
	}
	//En Kotlin
	class Car{
	var brand: String = “”
	var model: String = “”
	var year: Int = 0
	}

comparación de código Propiedades

Lo primero que notamos con la propiedades en Kotlin es que deben ser inicializadas con un valor por defecto

Kotlin nos generará su método getter y su setter de cada propiedad en en caso de se mutable y en el caso de ser inmutable solo tendremos acceso a ella (getter) pero no podemos modificarla mas adelante.

En Kotlin no habría que hacer ningún cambio adicional. Ya aquí comenzamos a notar lo conciso que es Kotlin.

comparación de código Propiedades

Su contraparte en Java seria definiendo las propiedades como privadas y generamos los métodos getters y setters respectivos quedando de la siguiente forma.


	//En java
	public class Car {
	private String brand;
	private String model;
	private int year;

	public String getBrand() {
	return brand;
	}

	public void setBrand(String brand) {
	this.brand = brand;
	}

	public String getModel() {
	return model;
	}

	public void setModel(String model) {
	this.model = model;
	}

	public int getYear() {
	return year;
	}

	public void setYear(int year) {
	this.year = year;
	}

comparación de código // Metodos

En Kotlin para definir un método usamos la palabra reservada fun y la definimos como una función.



	//En Java
	public class Car {
	//propiedades omitidas para este ejemplo

	public boolean isRecent(){
	return this.year >= 2015
	}
	}


	//En Kotlin
	class Car{
	//propiedades omitidas para este ejemplo

	fun isRecent(): Boolean {
	return this.year >= 2015
	}
	// otra forma de expresar la función
	fun isRecent() = this.year >= 2015
	}

comparación de código // Constructor

En java usamos un método public con el nombre de la clase

En Kotlin el constructor se define con la palabra reservada constructor y se define justo después del nombre de la clase.



	public class Car {
	//propiedades

	public Car(String brand, String model, int year) {
	this.brand = brand;
	this.model = model;
	this.year = year;
	}

	//Getters and Setters
	}

	class Car constructor(brand: String, model: String, year: Int){
	val brand: String = brand. //Asignamos el valor recibido por el constructor
	val model: String = model //Al ser inmutable solo nos generara el getter.
	var year: Int = year //Esta propiedad tendrá su getter y setter

	//metodos definidos …

	}

comparación de código // Constructor

Importante en Kotlin ,si el constructor primario (se pueden tener varios constructores) es publico se puede omitir la palabra reservada constructor del método constructor primario,y si dentro

de los argumentos del constructor definimos si declaramos que sean propiedades podemos omitir las mismas dentro del cuerpo de la clase.


	class Car (val brand: String, val model: String, var year: Int){

	//Ya no hay necesidad de definir nuevamente las propiedades 

	fun isRecent() = this.year >= 2015
	}
